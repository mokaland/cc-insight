# /記録 ワークフロー

## 使い方
```
/記録
```

## 手順

1. **プロジェクト確認**
   - 現在のプロジェクトを特定
   - `.agent/PROJECT_CONTEXT.md` を読み込む

2. **作業内容を整理**
   - 本日修正したバグ
   - 追加した機能
   - 変更したファイル

3. **PROJECT_CONTEXT.md を更新**
   - 作業履歴セクションに本日の内容を追記
   - 新しいCron Jobがあれば一覧に追加
   - 設計意図の変更があれば主要機能セクションを更新

4. **コミット**
   ```bash
   git add .agent/PROJECT_CONTEXT.md
   git commit -m "docs: Update PROJECT_CONTEXT.md with today's work"
   git push
   ```

5. **完了報告**
   - 「✅ 今日の作業を記録しました」
