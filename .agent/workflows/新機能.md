# /新機能 ワークフロー

## 使い方
```
/新機能 [機能の説明]
```

## 手順

1. **プロジェクト確認**
   - `.agent/PROJECT_CONTEXT.md` を読み込む
   - 現在のブランチを確認（staging想定）

2. **要件整理**
   - 機能の目的を明確化
   - 影響範囲を調査

3. **計画作成**
   - 変更するファイル一覧
   - 実装順序
   - 菅原さんに確認「この方針で進めますか？」

4. **実装**
   - 承認後、段階的に実装
   - 各段階でビルド確認

5. **完了報告**
   - 変更サマリーを報告
   - 「/反映 でstagingにデプロイできます」と案内
