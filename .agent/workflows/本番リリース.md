# /本番リリース ワークフロー

## 使い方
```
/本番リリース
```

## 手順

1. **事前確認**
   - staging環境で動作確認済みか確認
   - 「staging環境で確認済みですか？」

2. **マージ準備**
   - 変更内容のサマリーを作成
   - 菅原さんに確認「以下の変更をmainにマージしますか？」

3. **許可後のみ実行**
   ```bash
   git checkout main
   git merge staging
   git push
   git checkout staging
   ```

4. **完了報告**
   - 「✅ mainにマージしました」
   - 「本番環境（会員が使える環境）に反映されます」

## ⚠️ 重要
- 許可なしにマージしない
- 会員に影響するため慎重に確認
