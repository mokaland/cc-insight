# /反映 ワークフロー

## 使い方
```
/反映
```

## 手順

1. **ビルド確認**
   ```bash
   npm run build
   ```

2. **ビルド成功の場合のみ続行**

3. **コミット＆プッシュ**
   ```bash
   git add -A
   git commit -m "[適切なコミットメッセージ]"
   git push
   ```

4. **完了報告**
   - 「✅ stagingにデプロイしました」
   - 「staging環境で動作確認してください」
   - 「本番に反映する場合は /本番リリース を使用」

## 注意
- これはstagingへのデプロイ（菅原さんのみ確認可能）
- 会員に見える本番には反映されない
