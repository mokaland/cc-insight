# /バグ修正 ワークフロー

## 使い方
```
/バグ修正 [問題の説明]
```

## 手順

1. **状況確認**
   - エラーメッセージや症状を確認
   - 再現手順を把握

2. **原因調査**
   - 関連ファイルを調査
   - ログやコンソールエラーを確認

3. **修正**
   - 最小限の変更で修正
   - 他の機能に影響がないことを確認

4. **ビルド確認**
   - `npm run build` で確認

5. **完了報告**
   - 原因と修正内容を報告
   - 「/反映 でstagingにデプロイできます」と案内
